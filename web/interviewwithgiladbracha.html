There’s people using it, there’s people, even before we had 1.0, there’s people doing startups with it, which is rather brave of them given that we were changing things under their feet all the time. Definitely we expect to see considerable update. The language has been designed to be kind of comfortable for people. It’s very familiar, if you come from the curly-brace tradition, then you should be able to get the basic idea without any surprises.

They introduce a delay of these compilations, which can be quite slow at times and so forth. Our development environment basically uses a build of Chrome, of Chromium basically, that already has Dart in it, we call it Dartium. You get instantaneous response, just like with a dynamic language, like any other dynamic language like Ruby or JavaScript or something, so your development cycle is really quick.

I’m Java background, did a lot of Ruby lately. It’s hard to go back to that long build cycle.

Basically, a lot of us are old Smalltalkers at heart, and we had the short build cycle decades ago and the rest of the world looked strangely on us. Slowly these ideas are getting into the mainstream.
